Buttons

Every button is automatically a cursor target (no attribute needed) — hovering an outline/ghost button (fill) hides its own border/background and lets the cursor morph into its exact shape instead, staying the plain white ring (mix-blend-mode: exclusion) — it blends with, rather than covers, whatever's underneath, so the label stays visible throughout. Everything else just scales the dot (scale, the default). The button itself is never recolored either way. Primary is the neutral, high-contrast CTA (foreground-on-background); Accent is the rarer, branded highlight color, reserved for standing out rather than every button's default.

@for (variant of variants; track variant) { }

Tone — bordered destructive

tone is a modifier independent of variant — outline + tone="danger" is the one, canonical "bordered destructive" look (border width always matches every other bordered button — one shared token, never a per-variant guess). Its red text/border stay exactly as-is at rest and on hover; only the cursor morphs onto it, same as any other outline/ghost button.

Shape — rounded vs. pill

Google Design uses both squared (8px) and fully-rounded pill (100px) outline buttons, depending on context — shape controls this independently of variant. The cursor morph reads each button's actual border-radius, so it conforms to whichever shape is hovered rather than always becoming a fixed pill.

Inputs

Cards

Elevation (1-5) tints each card toward --cr-color-surface-tint (the accent hue, pale in light theme / white in dark) as the primary cue stays box-shadow — see the Materials section below for why cards stay opaque rather than using crMaterial. interactive adds the same tint, stronger, on hover — a presentational affordance for a card the caller is structuring as clickable. Materials

crMaterial is reserved for genuine translucent overlay/glass contexts floating above other content — menus, toasts, dropdowns — not for regular composable surfaces like Card, which stay opaque via the elevation tokens above: a card needs to stay legible regardless of what's behind it, and compounding transparency breaks down once surfaces stack on surfaces (a card on a sheet on a page). Each tier tints from --cr-color-surface-tint at increasing opacity, with blur growing alongside it (except ultra-thin, which is tint-only, no blur) — shown twice: over a photo, so the blur itself is legible in both themes, and over the plain --cr-color-surface base, so the tint alone (the part that still matters when there's nothing busy behind it to blur, e.g. a toast on a plain page) is legible too. Each tier's name sits in a small opaque chip (independent of the tier's own translucency, so the label always reads).

Shapes

crShape/crShapeHover clip an element into a basic shape via border-radius, with an optional distinct shape to morph into under the pointer — a plain CSS :hover transition, no cursor involvement. The custom cursor stays its normal small dot; only the image itself animates.
