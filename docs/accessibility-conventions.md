# Accessibility conventions

House style for `libs/shared/ui`: `data-*` is a pure styling hook with zero assistive-technology (AT) meaning; `aria-*` carries all AT-relevant state. CSS must never select on an `aria-*` attribute — style off `data-*` only, even when the two are mirrored from the same source signal.

| Attribute pattern                                           | Purpose                                                                       | When to use                                                                                                                                                                     | Reference                                                                                    |
| ----------------------------------------------------------- | ----------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------- |
| `data-variant` / `data-size` / `data-shape` / `data-tone`   | Visual variant selection, zero AT meaning                                     | Any component with mutually-exclusive style variants                                                                                                                            | `button[data-variant='primary']`                                                             |
| `data-elevation` / `data-padded`                            | Layout/visual state, zero AT meaning                                          | Boolean or scalar style toggles (empty-string-or-null for booleans)                                                                                                             | `cr-card[data-elevation='2']`                                                                |
| `data-invalid` **paired with** `aria-invalid`               | Style hook mirrored 1:1 from a real ARIA attribute, single source signal      | Whenever a styling data-attribute has AT-relevant meaning — always pair, driven from one input/signal                                                                           | `Input` (`libs/shared/ui/src/lib/input/input.ts`) — canonical template, reuse verbatim       |
| `data-cr-cursor-*`                                          | Cross-library styling contract (`ui` ↔ `cursor` libs)                         | Internal only, never a public component API surface                                                                                                                             | `--cr-cursor-fill-color` consumers                                                           |
| `role="status"` / `aria-live`                               | Only for content that _dynamically updates_ and should be announced           | Opt-in only — never on static content                                                                                                                                           | `Badge`'s `live` input (`libs/shared/ui/src/lib/badge/badge.ts`)                             |
| `aria-busy` **paired with** `aria-disabled` + a click guard | Signals ongoing content mutation and blocks activation while busy             | `aria-busy` alone does not block activation (WAI-ARIA APG) — pair with `aria-disabled` and prevent the click, don't touch the native `disabled` attribute if the caller owns it | `Button`'s `loading` input (`libs/shared/ui/src/lib/button/button.ts`)                       |
| `aria-pressed` (stable label across states)                 | Two-state toggle buttons; visible label text must stay constant across states | Icon-only or icon-plus-label toggle buttons                                                                                                                                     | `ThemeToggle` (`libs/shared/ui/src/lib/theme-toggle/theme-toggle.html`) — canonical template |
| `aria-describedby`                                          | Associates an input with its validation/help message                          | Whenever a message element (error, hint) exists alongside a form control                                                                                                        | `Input`'s `errorId` input                                                                    |
| `:host(:focus-visible)` → `var(--cr-color-focus-ring)`      | Focus ring on keyboard/AT focus only, ≥3:1 contrast (WCAG 1.4.11)             | Every interactive component, always via `:focus-visible`, never bare `:focus`                                                                                                   | `button.css` / `input.css` — mandatory standing rule                                         |
| `min-block-size` touch-target floor (36/40/48px by size)    | WCAG 2.5.8 (AA) minimum target size                                           | Every interactive component's `sm`/`md`/`lg` size variants                                                                                                                      | `button.css` / `input.css`                                                                   |

## Non-interactive by default

`Card` (`libs/shared/ui/src/lib/card/card.ts`) has no `role` or `tabindex` — it is not interactive, and any interactive affordance inside it (a `Book now` button) is its own real focusable element. If a future clickable-card variant is built, the host itself must become a real `<a>` or `<button>` (or wrap one), never a `<div>` with `role="button"` + `tabindex="0"` bolted on — that anti-pattern re-implements native keyboard activation, focus, and AT semantics by hand and reliably gets at least one of them wrong.

## Canonical templates to copy from

- **`data-invalid`/`aria-invalid` pairing:** `Input` (`libs/shared/ui/src/lib/input/input.ts`).
- **`aria-pressed` toggle:** `ThemeToggle` (`libs/shared/ui/src/lib/theme-toggle/theme-toggle.html`).
- **Opt-in live region:** `Badge`'s `live` input (`libs/shared/ui/src/lib/badge/badge.ts`).
- **`aria-busy` + activation guard:** `Button`'s `loading` input (`libs/shared/ui/src/lib/button/button.ts`).
